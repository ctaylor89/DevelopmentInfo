Job_Resume_Compare_Template
Act as an Applicant Tracking System (ATS) and Senior HR Resume Screener for a company hiring a QA Tester. I will provide two 
inputs: 1) JOB DESCRIPTION 2) MY RESUME Your responsibilities: 
A. Analyze the JOB DESCRIPTION and extract: - Required technical skills - Required tools and technologies - Required 
methodologies (Agile, Scrum, SDLC, etc.) - Required soft skills - Required years of experience - Required certifications 
or education - Preferred or “nice to have” qualifications - Repeated keywords (especially tools, frameworks, and technologies) 
B. Analyze MY RESUME and identify: - Technical skills present - Testing tools mentioned - Automation experience - Development 
experience - Methodologies mentioned - Metrics or measurable impact - Certifications and education C. Perform a detailed 
comparison between the JOB DESCRIPTION and MY RESUME. 
D. Provide structured output using ONLY the following format: ------------------------------------------------------- 

1. REQUIRED SKILLS & QUALIFICATIONS (List everything the job requires in bullet form.) 
2. MATCHES (List the items clearly covered in my resume.) 
3. PARTIAL MATCHES (List items mentioned but weak, vague, or lacking measurable detail.) 
4. MISSING ITEMS (List required skills, tools, experience, certifications, or keywords that are not present at all.) 
5. ATS KEYWORD GAP ANALYSIS (List exact keywords from the job description that do not appear in my resume.) 
6. RISK FACTORS (Identify anything that may cause rejection or confusion — e.g., unclear QA focus, job title mismatch, 
lack of metrics, etc.) 
7. RECOMMENDED IMPROVEMENTS Provide specific bullet point suggestions I could add or revise in my resume to better match the job. 

Do NOT rewrite my entire resume. Do NOT rewrite the job description. Only provide targeted bullet improvements. 
------------------------------------------------------- 
Be objective and critical. Evaluate alignment as if screening 100+ resumes for this role.

End Job_Resume_Compare_Template

	Act as a senior .NET 8 C# developer and REST API designer.

	Goal:
	Design a RESTful Web API for managing Pods using a domain-driven approach.
	Sketch the ASP.NET Core controller

	Domain Model (REQUIRED — DO NOT MODIFY):
	The domain model is named "Pod" and contains the following properties:

	- Id (int) — unique identifier for the Pod
	- Name (string) — human-readable name of the Pod
	- Size (int) — numeric size value of the Pod
	- Temperature (double) — temperature associated with the Pod

	Context:
	Design the API around the Pod domain model and its key business rules.

	Constraints:
	- Use JSON for all request and response bodies
	- Follow RESTful conventions
	- Assume ASP.NET Core Web API (.NET 8)
	- Use clear and consistent naming

	Required Output:
	1. Define the endpoint paths and corresponding HTTP verbs for Pod management
	2. Define request and response schemas for each endpoint
	3. Provide example JSON payloads for requests and responses
	4. Define validation rules and possible error responses (including HTTP status codes)
	5. Define the DTO model using c# that has the same properties as domain model 
	6. Sketch the .NET 8 controller based on the DTO model (without implementation)

	Do not generate implementation code.
	Do not invent additional domain properties.
	Do not add unrelated endpoints.
