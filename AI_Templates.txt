Job_Resume_Compare_Template
Act as an Applicant Tracking System (ATS) and Senior HR Resume Screener for a company hiring a QA Tester. I will provide two 
inputs: 1) JOB DESCRIPTION 2) MY RESUME Your responsibilities: 
A. Analyze the JOB DESCRIPTION and extract: - Required technical skills - Required tools and technologies - Required 
methodologies (Agile, Scrum, SDLC, etc.) - Required soft skills - Required years of experience - Required certifications 
or education - Preferred or “nice to have” qualifications - Repeated keywords (especially tools, frameworks, and technologies) 
B. Analyze MY RESUME and identify: - Technical skills present - Testing tools mentioned - Automation experience - Development 
experience - Methodologies mentioned - Metrics or measurable impact - Certifications and education C. Perform a detailed 
comparison between the JOB DESCRIPTION and MY RESUME. 
D. Provide structured output using ONLY the following format: ------------------------------------------------------- 

1. REQUIRED SKILLS & QUALIFICATIONS (List everything the job requires in bullet form.) 
2. MATCHES (List the items clearly covered in my resume.) 
3. PARTIAL MATCHES (List items mentioned but weak, vague, or lacking measurable detail.) 
4. MISSING ITEMS (List required skills, tools, experience, certifications, or keywords that are not present at all.) 
5. ATS KEYWORD GAP ANALYSIS (List exact keywords from the job description that do not appear in my resume.) 
6. RISK FACTORS (Identify anything that may cause rejection or confusion — e.g., unclear QA focus, job title mismatch, 
lack of metrics, etc.) 
7. RECOMMENDED IMPROVEMENTS Provide specific bullet point suggestions I could add or revise in my resume to better match the job. 

Do NOT rewrite my entire resume. Do NOT rewrite the job description. Only provide targeted bullet improvements. 
------------------------------------------------------- 
Be objective and critical. Evaluate alignment as if screening 100+ resumes for this role.

End Job_Resume_Compare_Template

-----------------------------------------------------------
Act as a senior .NET 8 C# developer and REST API designer.
Goal: Provide step-by-step instructions to add authentication to an existing ASP.NET Core Web API application
that is not currently using any authentication mechanism.

1. Choose an authentication method:
   - Decide between JWT Bearer tokens, cookie-based authentication, or another method based on your application's needs.

2. Install necessary NuGet packages:
   - For JWT Bearer authentication, install the `Microsoft.AspNetCore.Authentication.JwtBearer` package.
   - For cookie-based authentication, install the `Microsoft.AspNetCore.Authentication.Cookies` package.

3. Configure authentication services:
   - In `Program.cs`, add the authentication services to the DI container:
     ```csharp
     builder.Services.AddAuthentication(options =>
     {
         options.DefaultAuthenticateScheme = JwtBearerDefaults.AuthenticationScheme;
         options.DefaultChallengeScheme = JwtBearerDefaults.AuthenticationScheme;
     })
     .AddJwtBearer(options =>
     {
         options.TokenValidationParameters = new TokenValidationParameters
         {
             ValidateIssuer = true,
             ValidateAudience = true,
             ValidateLifetime = true,
             ValidateIssuerSigningKey = true,
             // Set your issuer, audience, and signing key here
         };
     });
     ```

4. Protect your API endpoints:
   - Use the `[Authorize]` attribute on controllers or actions that require authentication.
   - Example:
     ```csharp
     [Authorize]
     [ApiController]
     [Route("api/[controller]")]
     public class PodController : ControllerBase
     {
         // Your actions here
     }
     ```

5. Implement user login and token generation:
   - Create a login endpoint that validates user credentials and generates a JWT token.
   - Example:
     ```csharp
     [HttpPost("login")]
     public IActionResult Login([FromBody] LoginRequest request)
     {
         // Validate user credentials
         // Generate JWT token
         return Ok(new { Token = jwtToken });
     }
     ```

6. Test your authentication:
   - Use a tool like Postman to test your API endpoints.
   - Include the JWT token in the `Authorization` header for protected endpoints.

7. (Optional) Implement user registration:
   - Create a registration endpoint to allow new users to sign up.
   - Hash passwords and store user information securely.

8. (Optional) Configure authorization policies:
   - Define custom authorization policies if needed.
   - Use the `[Authorize(Policy = "YourPolicyName")]` attribute to apply them.

9. (Optional) Add role-based or claims-based authorization:
   - Implement role-based or claims-based authorization as needed for your application.

10. Document your API:
    - Update your API documentation to include authentication details.
    - Provide examples of how to obtain and use JWT tokens or cookies.
